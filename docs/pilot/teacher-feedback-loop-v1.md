# Teacher Feedback Loop v1

## Purpose

Tendo should improve based on real teacher and learner use, not guesses.

This feedback loop is designed for early P7 pilots.

---

## Feedback principles

1. Ask about usefulness, not just liking.
2. Ask teachers what saved time.
3. Ask learners what helped understanding.
4. Record exact quotes when possible.
5. Look for repeated patterns across teachers.
6. Separate urgent blockers from nice-to-have ideas.

---

## Feedback categories

### 1. Content clarity

Ask:

- Was the Big idea clear?
- Were Key words understandable?
- Was Learn it in parts easy to follow?
- Was the PLE tip useful?
- Was anything inaccurate or too simplified?

### 2. Visual usefulness

Ask:

- Did the diagram help you teach/explain?
- Did learners understand the diagram?
- Was any visual confusing?
- Which topic needs a better diagram?

### 3. Practice and quizzes

Ask:

- Were quiz questions too easy, hard or balanced?
- Did explanations help after wrong answers?
- Which topics need more practice questions?
- Are questions PLE-like enough?

### 4. Worksheet generator

Ask:

- Would you print these worksheets?
- Is the answer key useful?
- Are marks/time/class fields enough?
- What worksheet format would be better?
- Would mixed-subject worksheets help?

### 5. Classroom workflow

Ask:

- How would you use this in a real week?
- Would learners use it alone or teacher-led?
- What device setup is realistic?
- What would stop you from using it?

### 6. Commercial signal

Ask carefully:

- Would this save teacher preparation time?
- Would the school want this for P7 revision?
- Who would decide whether to adopt it?
- What would make it worth paying for?

## How to prioritise feedback

### Fix immediately if:

- content is wrong
- wording is inappropriate
- teacher cannot navigate
- worksheet is unusable
- learners misunderstand a diagram

### Prioritise soon if:

- several teachers ask for the same feature
- a topic is important for PLE and weak in class
- a diagram is repeatedly requested
- worksheet formatting blocks classroom use

### Defer if:

- it is a nice feature but not needed for pilot
- it requires a backend/login
- it is about videos before content/workflow is proven
- only one person requests it and it does not affect learning

---

## Feedback review rhythm

During pilot:

- Collect notes after every school visit.
- Summarise feedback once per week.
- Pick 1–3 improvements only.
- Push grouped fixes, not tiny changes every day.

---

## Founder decision after 2–3 school touchpoints

After a few real teacher conversations, decide:

1. Which subject has strongest pull?
2. Is worksheet generation a strong teacher hook?
3. Do teachers care more about learner practice or teacher prep?
4. Is P7 enough for first sale, or is P6 needed quickly?
5. What is the simplest paid pilot offer?
